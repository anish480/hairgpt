The multi-attribute challenge

The current LLM classifier predicts 7 attributes simultaneously (hair_pattern, pattern_code, frizz, damage, length, confidence, notes). A custom ML model would need:

  • Separate classification heads for each attribute (or a multi-task learning setup)
  • Separate labeling for every attribute on every image — 800 images x 6 discrete attributes = 4,800 labels minimum
  • No "notes" equivalent — a vision CNN cannot generate free-text observations like "highlights visible, some heat damage at ends"
  • No "confidence" reasoning — model confidence (softmax probability) is not the same as the LLM's reasoned self-assessment

4. Side-by-Side Comparison

Gemini 2.5 Flash (Current)

Pros

  • Zero training data required — works out of the box
  • Classifies all 7 attributes in a single call
  • Generates free-text "notes" with nuanced observations
  • Self-assesses confidence with reasoning
  • Handles edge cases gracefully (blurry photos, non-hair images)
  • No model hosting infrastructure — pure API call
  • Automatically improves as Google updates the model
  • Structured JSON output guaranteed by schema enforcement
  • Instant deployment — already integrated and working
  • Handles diverse hair types, lighting conditions, angles without training

Cons

  • Vendor dependency on Google / Vertex AI
  • Per-request cost increases linearly with traffic
  • Latency ~1–3s per request (network round-trip to Vertex AI)
  • Non-deterministic — same image may yield slightly different results
  • Cannot be customized to Moxie's specific hair type taxonomy
  • Potential for hallucinated observations in "notes" field
Custom ML Model (Proposed)

Pros

  • Fixed hosting cost at scale — does not increase per-request
  • Much faster inference (~50–200ms on GPU, ~200–500ms on CPU)
  • Fully deterministic outputs
  • Can be customized to Moxie's exact taxonomy
  • No vendor dependency for inference once deployed
  • Can run on-device (TFLite/ONNX) for zero-latency classification
  • Model is an owned asset

Cons

  • Requires ~800+ labeled images across ALL attributes (currently pending)
  • Cannot generate free-text notes or nuanced observations
  • 6 separate classification heads needed for multi-attribute output
  • 800 images is marginal — expect lower accuracy on rare categories (coily, 4a-4c)
  • Hosting cost even when idle (~$800/mo for Vertex AI GPU endpoint, or ~$490/mo for Cloud Run L4 GPU)
  • Requires ML engineering effort to build, train, validate, and deploy
  • Ongoing maintenance: retraining pipeline, data drift monitoring, model versioning
  • Does not handle edge cases well (non-hair images, blurry photos) without explicit training
  • Training data class imbalance likely (wavy/curly overrepresented in Indian market)

9. The 800-Image Dataset Constraint

Critical Constraint

The Moxie team has ~800 labeled hair photos pending. This is a key factor in the decision:

ConsiderationAssessment
Is 800 images enough for fine-tuning? Marginal. Published research shows fine-tuned vision models (ResNet, EfficientNet) can achieve 75–85% accuracy on hair-type classification with similar dataset sizes. However, this is for a single classification task. Multi-attribute classification across 6 fields with 800 images will perform inconsistently.
Class balance Likely imbalanced. Moxie's customer base skews toward wavy/curly hair (2a–3c) in the Indian market. Straight (1a–1c) and coily (4a–4c) categories will likely have far fewer examples, leading to poor classification on those types.
Labeling quality Uncertain. Hair type labels are subjective, especially for sub-codes (2b vs 2c). Inter-annotator agreement on frizz and damage levels is typically low. The quality of the 800 labels will directly bound model quality.
Data augmentation potential Moderate. Standard augmentations (flip, rotate, color jitter, crop) can synthetically expand the dataset 5–10x, but cannot compensate for missing categories or labeling inconsistency.
Scaling data collection Slow path. Growing from 800 to 5,000+ high-quality labeled images would take months. Each image needs expert labeling across all 6 attributes.

10. Hybrid Approach (Future Option)

Hybrid: LLM + ML

A phased approach could capture the best of both worlds:

PhaseApproachWhen
Phase 1 (Now) Keep Gemini 2.5 Flash. Use it to auto-label the 800 photos (and future uploads) to build a high-quality labeled dataset at near-zero marginal cost. Log every classification with the original image for future training data. Immediate
Phase 2 (3–6 months) Once 3,000–5,000 LLM-labeled + human-verified images exist, train a lightweight EfficientNet multi-head classifier for the 4 most objective attributes (hair_pattern, pattern_code, length, frizz_level). Keep Gemini for "damage_signs" and "notes". When dataset reaches 3K+ verified images
Phase 3 (6–12 months) If traffic exceeds 10K/day and cost becomes a concern, deploy the custom model on Cloud Run with scale-to-zero. Use Gemini as a fallback for edge cases and to generate "notes". When daily volume justifies infrastructure cost

11. Recommendation

Recommendation

Keep Gemini 2.5 Flash for now. Start building the dataset for a future transition.

Reasoning:

  1. Cost is not a problem at current scale. At 100–1,000 classifications/day, Gemini costs $2.64–$26.40/month. A custom ML endpoint would cost $490–$800/month minimum, even with Cloud Run scale-to-zero. The breakeven point is somewhere around 25,000–30,000 classifications per day.
  2. 800 images is insufficient for a 7-attribute classifier. The current LLM approach handles all 7 attributes plus free-text notes. A custom ML model would need to drop the "notes" field entirely and would struggle with the 12-class hair pattern sub-code problem given the dataset size.
  3. The LLM is already integrated and working. Switching introduces engineering cost (2–4 weeks development + ongoing maintenance) with no clear accuracy improvement and a definite loss of the "notes" and confidence-reasoning capabilities.
  4. Latency is acceptable. The 1–3s classification time is not the user-facing bottleneck — the subsequent chat LLM call takes longer.
  5. Start accumulating data now. Log every Gemini classification alongside the original image. Use the LLM as an auto-labeler. When the dataset grows to 3,000–5,000 verified images and traffic justifies the infrastructure spend, re-evaluate the custom ML path.
